Introduction

MarathonITE uses Selenium WebDriver to connect to various browsers. Before you can use MarathonITE for recording and playing your tests, you need to setup your test machine. The following sections explain in detail the steps required to make a browser available for testing with MarathonITE. You can refer to this help message any time from the Browser menu.

Advanced Configuration

You can provide advanced configuration through the fixture code of Marathon test scripts. Add a method as following to your fixture class.

def desired_capabilities(caps)
   # Add any extra capabilities you require into the caps dict
   caps
end

The extra capabilities will be passed to the corresponding webdriver instance. For example, you can define the Firefox binary path while using Marionette driver by adding the following: 

 caps['moz:firefoxOptions'] = { :binary => '/usr/lib/myfirefox/firefox' }

Refer to documentation of specific webdriver implementations for details of options that are available.

 

Chrome

MarathonITE supports both recording and playback using the Chrome browser. Please use the latest versions of Chrome browser and chromedriver with MarathonITE.

  1. If it is not already downloaded, download and install google chrome for your platform from: https://www.google.com/chrome/.
  2. Download chromedriver from https://sites.google.com/a/chromium.org/chromedriver/. chromedriver provides the Selenium/WebDriver bindings for chrome browser.
  3. Place the chromedriver executable in a folder that is in your PATH. For Windows systems, C:\Windows is a good option. Alternatively, you can keep the chromedriver executable in any folder and add the folder to your PATH. Refer to the operating system instructions to add the folder to your PATH.

You can customize the chromedriver behavior by defining the following environment variables.

Variable Description
webdriver.chrome.driver System property that defines the location of the chromedriver executable that will be used by MarathonITE.
webdriver.chrome.logfile System property that defines the location of the log that will be written.
webdriver.chrome.verboseLogging Boolean system property that defines whether the chromedriver executable should be started with verbose logging.
webdriver.chrome.silentOutput Boolean system property that defines whether the chromedriver executable should be started in silent mode.
webdriver.chrome.whitelistedIps System property that defines comma-separated list of remote IPv4 addresses which are allowed to connect to chromedriver.

Refer to chromedriver documentation for further details about these arguments.

 

Firefox

Use this driver for Firefox version 48 or later.
MarathonITE supports both recording and playback using the Firefox browser.

  1. If it is not already downloaded, download and install Firefox for your platform from: https://www.mozilla.org/en-US/firefox/new/.
  2. Download Marionette Driver from https://github.com/mozilla/geckodriver/releases. Marionette provides the Selenium/WebDriver bindings for firefox browser.
  3. Place the Marionette executable in a folder that is in your PATH. For Windows systems, C:\Windows is a good option. Alternatively, you can keep the Marionette executable in any folder and add the folder to your PATH. Refer to the operating system instructions to add the folder to your PATH.

You can customize the webdriver behavior by defining the following environment variables.

Variable Description
webdriver.gecko.driver System property that defines the location of the GeckoDriver executable that will be used by MarathonITE.

Refer to Marionette Driver documentation for further details.

 

Edge

MarathonITE supports both recording and playback using the Microsoft Edge browser. Please use the latest versions of Windows 10 Edge browser and corresponding edge driver executable with MarathonITE.

  1. Download MicrosoftWebDriver from https://developer.microsoft.com/en-us/microsoft-edge/tools/webdriver/. MicroSoftWebDriver provides the Selenium/WebDriver bindings for Edge browser. Note that your Windows 10 build should match the version of MicrosoftWebDriver you use. Please look at the documentation on the link for more details.
  2. Place the MicrosoftWebDriver executable in a folder that is in your PATH. For Windows systems, C:\Windows is a good option. Alternatively, you can keep the chromedriver executable in any folder and add the folder to your PATH. Refer to the operating system instructions to add the folder to your PATH.
  3. From an administrator command prompt run the following command:
    CheckNetIsolation LoopbackExempt -a -n=Microsoft.MicrosoftEdge_8wekyb3d8bbwe
    This command allows Edge driver to be connected to by MarathonITE.

You can customize the MicroSoftWebDriver behavior by defining the following environment variables.

Variable Description
webdriver.edge.driver System property that defines the location of the MicrosoftWebDriver executable that will be used by the MarathonITE.

Refer to MicrosoftWebDriver documentation for further details.

 

IE

MarathonITE supports only playback using the Internet Explorer browser. Please use the latest versions of Internet Explorer browser and iedriver with MarathonITE.

  1. Download iedriver from http://www.seleniumhq.org/download/. iedriver provides the Selenium/WebDriver bindings for Internet Explorer.
  2. Place the iedriver executable in a folder that is in your PATH. For Windows systems, C:\Windows is a good option. Alternatively, you can keep the iedriver executable in any folder and add the folder to your PATH. Refer to the operating system instructions to add the folder to your PATH.

You can customize the iedriver behavior by defining the following environment variables.

Variable Description
webdriver.ie.driver System property that defines the location of the IEDriverServer executable that will be used by MarathonITE.
webdriver.ie.driver.logfile System property that defines the location of the file where IEDriverServer should write log messages to.
webdriver.ie.driver.loglevel System property that defines the detalization level the IEDriverServer logs.
webdriver.ie.driver.engine System property that defines the implementation of the driver engine to use.
webdriver.ie.driver.host System property that defines host to which will be bound IEDriverServer.
webdriver.ie.driver.extractpath System property that defines path to which will be extracted IEDriverServer library.
webdriver.ie.driver.silent System property that defines logging to stdout for IEDriverServer.

Refer to iedriver documentation for further details.

 

Safari

MarathonITE supports only playback using the Safari 10 and above on OSX.

There are no configuration parameters available through the environmental variables for Safari driver.
